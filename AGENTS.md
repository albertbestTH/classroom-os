# Classroom OS repository instructions

Before planning or implementing any change, read [the Classroom OS Product Bible](docs/product/classroom-os-product-bible.md). It is the repository's authoritative guide for product direction, workspace separation, architecture boundaries, UX, security, delivery standards, and established decisions.

If a task conflicts with the Product Bible, stop and report the conflict. Do not weaken tenant or classroom isolation, authorization, mobile-first teacher priorities, or privacy safeguards. Never add real student data, secrets, or biometric functionality without the explicit governance approvals defined there.

More specific `AGENTS.md` files may add local technical instructions but may not override the Product Bible.
